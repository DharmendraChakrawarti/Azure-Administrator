# 3.1 Automate Deployment Using ARM Templates or Bicep — AZ-104 Study Notes

---

## Overview

Azure Resource Manager (ARM) templates and Bicep provide **Infrastructure as Code (IaC)** for deploying Azure resources declaratively. You define the desired state, and Azure handles creating/updating resources to match.

> [!IMPORTANT]
> **Bicep** is the successor to ARM JSON templates. It compiles down to ARM templates but is easier to read and write. The AZ-104 exam tests both formats.

---

## 3.1.1 Azure Resource Manager (ARM) Overview

### What is ARM?
- The **deployment and management service** for Azure
- All requests (Portal, CLI, PowerShell, SDK, REST API) go through ARM
- Provides a **consistent management layer** across all tools

### ARM Benefits

| Benefit | Description |
|---------|-------------|
| **Declarative syntax** | Define *what* to deploy, not *how* |
| **Idempotent** | Same template = same result every time |
| **Dependency management** | ARM determines correct deployment order |
| **Validation** | Templates are validated before deployment |
| **Modular** | Linked/nested templates for reusability |
| **Preview changes** | What-if operation to preview changes |

## 3.1.2 ARM Template Structure

### JSON Template Format

```json
{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "storageName": {
      "type": "string",
      "minLength": 3,
      "maxLength": 24,
      "metadata": { "description": "Storage account name" }
    },
    "location": {
      "type": "string",
      "defaultValue": "[resourceGroup().location]"
    },
    "storageSKU": {
      "type": "string",
      "defaultValue": "Standard_LRS",
      "allowedValues": ["Standard_LRS", "Standard_GRS", "Standard_ZRS"]
    }
  },
  "variables": {
    "uniqueStorageName": "[concat(parameters('storageName'), uniqueString(resourceGroup().id))]"
  },
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2022-09-01",
      "name": "[variables('uniqueStorageName')]",
      "location": "[parameters('location')]",
      "sku": { "name": "[parameters('storageSKU')]" },
      "kind": "StorageV2",
      "properties": { "supportsHttpsTrafficOnly": true }
    }
  ],
  "outputs": {
    "storageEndpoint": {
      "type": "string",
      "value": "[reference(variables('uniqueStorageName')).primaryEndpoints.blob]"
    }
  }
}
```

### Template Sections

| Section | Required? | Description |
|---------|-----------|-------------|
| **$schema** | ✅ Yes | URL of the JSON schema |
| **contentVersion** | ✅ Yes | Version of the template |
| **parameters** | ❌ No | Values provided at deployment |
| **variables** | ❌ No | Reusable values constructed from parameters |
| **resources** | ✅ Yes | Azure resources to deploy |
| **outputs** | ❌ No | Values returned after deployment |
| **functions** | ❌ No | User-defined functions |

### Parameter Types
- `string`, `int`, `bool`, `object`, `array`, `secureString`, `secureObject`

### Common Template Functions

| Function | Description | Example |
|----------|-------------|---------|
| `concat()` | Concatenate strings | `[concat('prefix-', parameters('name'))]` |
| `uniqueString()` | Deterministic hash | `[uniqueString(resourceGroup().id)]` |
| `resourceGroup()` | Current resource group info | `[resourceGroup().location]` |
| `reference()` | Runtime state of a resource | `[reference('storageAccount').primaryEndpoints.blob]` |
| `subscription()` | Current subscription info | `[subscription().subscriptionId]` |

## 3.1.3 Bicep Templates

### What is Bicep?
- A **domain-specific language** for deploying Azure resources
- **Transpiles to ARM JSON** — fully compatible with ARM
- Easier syntax, better tooling, VS Code extension

### Bicep vs ARM JSON

| Feature | ARM JSON | Bicep |
|---------|----------|-------|
| **Syntax** | Verbose JSON | Clean, concise |
| **Module support** | Linked templates (complex) | Native modules (simple) |
| **Tooling** | Basic | VS Code extension, IntelliSense |
| **Learning curve** | Steeper | Easier |
| **Compilation** | Direct | Transpiles to ARM JSON |

### Bicep Example

```bicep
@description('Storage account name')
@minLength(3)
@maxLength(24)
param storageName string

@description('Location for resources')
param location string = resourceGroup().location

@allowed(['Standard_LRS', 'Standard_GRS', 'Standard_ZRS'])
param storageSKU string = 'Standard_LRS'

var uniqueStorageName = '${storageName}${uniqueString(resourceGroup().id)}'

resource storageAccount 'Microsoft.Storage/storageAccounts@2022-09-01' = {
  name: uniqueStorageName
  location: location
  sku: {
    name: storageSKU
  }
  kind: 'StorageV2'
  properties: {
    supportsHttpsTrafficOnly: true
  }
}

output storageEndpoint string = storageAccount.properties.primaryEndpoints.blob
```

### Bicep Modules
```bicep
// main.bicep
module stgModule 'modules/storage.bicep' = {
  name: 'storageDeployment'
  params: {
    storageName: 'mystorage'
    location: 'eastus'
  }
}

output endpoint string = stgModule.outputs.storageEndpoint
```

## 3.1.4 Deploy ARM/Bicep Templates

### Deployment Scopes

| Scope | Use Case |
|-------|----------|
| **Resource Group** | Most common — deploy resources to a RG |
| **Subscription** | Deploy resource groups, policies |
| **Management Group** | Deploy policies across subscriptions |
| **Tenant** | Deploy management groups |

### PowerShell Deployment
```powershell
# Deploy to resource group
New-AzResourceGroupDeployment -ResourceGroupName "myRG" `
  -TemplateFile "template.json" `
  -TemplateParameterFile "parameters.json"

# Deploy Bicep
New-AzResourceGroupDeployment -ResourceGroupName "myRG" `
  -TemplateFile "main.bicep" `
  -storageName "mystorage"

# What-if (preview changes)
New-AzResourceGroupDeployment -ResourceGroupName "myRG" `
  -TemplateFile "template.json" -WhatIf

# Deploy to subscription scope
New-AzSubscriptionDeployment -Location "eastus" `
  -TemplateFile "sub-template.json"
```

### Azure CLI Deployment
```bash
# Deploy to resource group
az deployment group create --resource-group myRG \
  --template-file template.json --parameters parameters.json

# Deploy Bicep
az deployment group create --resource-group myRG \
  --template-file main.bicep --parameters storageName=mystorage

# What-if
az deployment group what-if --resource-group myRG \
  --template-file template.json

# Validate template
az deployment group validate --resource-group myRG \
  --template-file template.json
```

### Deployment Modes

| Mode | Description |
|------|-------------|
| **Incremental** (default) | Leaves existing resources unchanged; adds/updates defined resources |
| **Complete** | Deletes resources not in template; adds/updates defined resources |

> [!CAUTION]
> **Complete mode** will **delete resources** that exist in the resource group but are NOT defined in the template. Always use **What-If** before deploying in complete mode.

### Parameter Files
```json
{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentParameters.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "storageName": { "value": "myuniquestorage" },
    "storageSKU": { "value": "Standard_GRS" }
  }
}
```

## 3.1.5 Export and Manage Templates

### Export Template from Azure
- **Resource group** → Automation → Export template
- **Resource** → Export template
- Exports current state as ARM JSON

### Template Specs
- Store ARM templates as a **resource type** in Azure
- **Versioned** and shareable across the organization
- Can be deployed like any other template

```powershell
# Create template spec
New-AzTemplateSpec -ResourceGroupName "myRG" `
  -Name "storageSpec" -Version "1.0" `
  -Location "eastus" -TemplateFile "template.json"

# Deploy from template spec
$specId = (Get-AzTemplateSpec -ResourceGroupName "myRG" -Name "storageSpec" -Version "1.0").Versions.Id
New-AzResourceGroupDeployment -ResourceGroupName "myRG" `
  -TemplateSpecId $specId
```

## Exam Tips — Section 3.1

- ARM template required sections: **$schema**, **contentVersion**, **resources**
- **Bicep** is cleaner syntax that transpiles to ARM JSON
- Know the deployment modes: **Incremental** (default, safe) vs. **Complete** (deletes extras)
- **What-If** previews changes without deploying
- Know common functions: `concat()`, `uniqueString()`, `resourceGroup()`, `reference()`
- **secureString** parameters protect sensitive values (not logged, not visible after deployment)
- **Template Specs** store versioned templates as Azure resources
- Deployment scopes: Resource Group > Subscription > Management Group > Tenant
- **Linked templates** (ARM) vs. **Modules** (Bicep) for modular deployments
- ARM validates templates **before** deployment begins
- **Idempotent** — deploying the same template again results in the same state
