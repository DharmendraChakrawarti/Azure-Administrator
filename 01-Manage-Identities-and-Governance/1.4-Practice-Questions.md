# 1.4 Practice Questions — Manage Identities and Governance

> Scenario-based questions to test your understanding. Answers are at the bottom.

---

## Questions

### Q1 — Dynamic Groups
Your company has 200 employees in the Marketing department. You need to automatically add all Marketing users to a security group when their department attribute is set. Which type of group membership should you configure, and what license is required?

---

### Q2 — Administrative Units
You are the Global Admin for Contoso. The Seattle office has its own helpdesk team that should only be able to reset passwords for Seattle users. They should NOT be able to reset passwords for any other office. What should you configure?

---

### Q3 — RBAC Role Assignment
A developer named Alex needs to be able to create and manage VMs in the "Dev-RG" resource group, but should NOT be able to manage network resources or assign roles to other users. Which built-in role should you assign and at what scope?

---

### Q4 — Custom RBAC Role
Your security team needs a custom role that allows users to start, stop, and restart VMs but not delete them. The role should only be assignable in the "Production" subscription. Which properties do you need to configure in the custom role definition?

---

### Q5 — Azure Policy
You need to ensure that all resources deployed in the "Production-RG" resource group have a "CostCenter" tag. Resources without this tag should be denied. Which Azure Policy effect should you use, and what is the policy definition name?

---

### Q6 — Resource Locks
A production storage account has a **Delete** lock applied. A developer tries to modify the storage account's replication setting from LRS to GRS. Will this succeed? What if the lock type was **ReadOnly**?

---

### Q7 — Management Groups
Your organization has 50 subscriptions that need to comply with a policy restricting deployment to US regions only. You want to apply this policy once and have it affect all subscriptions. What should you do?

---

### Q8 — Guest Users
You need to invite an external partner (partner@external.com) to collaborate on a project. The partner should be able to access specific applications but NOT enumerate the directory. What type of account should you create, and what are the default permissions?

---

### Q9 — SSPR Configuration
You are configuring Self-Service Password Reset for your organization. Your CTO asks why admin accounts require two authentication methods even though the policy is set to require only one. What is the explanation?

---

### Q10 — Subscription Transfer
Your company is transferring a subscription from Tenant A to Tenant B. The development team is worried about losing their RBAC role assignments and Azure Policy assignments. What should you tell them?

---

### Q11 — Policy vs RBAC
A junior admin is confused about when to use Azure Policy versus RBAC. They want to prevent anyone from creating VMs larger than Standard_D4s_v3. Should they use Azure Policy or RBAC?

---

### Q12 — Deny Assignments
A team lead says they want to create a deny assignment to block a specific user from deleting VMs. Can they create this deny assignment directly? How are deny assignments created?

---

### Q13 — Tag Inheritance
You've applied the tag `Environment=Production` on resource group "Prod-RG". A new VM is created in this RG without any tags. Does the VM inherit the tag from the resource group?

---

### Q14 — Bulk Operations
You need to create 500 user accounts in Azure AD. The user data is in a spreadsheet. What is the most efficient method?

---

### Q15 — Scope Inheritance
User Sarah is assigned the **Reader** role at the subscription level and the **Contributor** role at the "Dev-RG" resource group level. What are her effective permissions on a VM inside "Dev-RG"? What about a VM in "Prod-RG"?

---

## Answers

### A1
Configure a **Dynamic User** membership type with the rule `(user.department -eq "Marketing")`. Requires **Azure AD Premium P1 or P2** license.

### A2
Create an **Administrative Unit** named "Seattle Office", add all Seattle users to it, then assign the **Password Administrator** role scoped to that AU to the Seattle helpdesk team. Requires **Azure AD Premium P1** for the admins managing the AU.

### A3
Assign the **Virtual Machine Contributor** role at the **resource group scope** (`/subscriptions/{sub-id}/resourceGroups/Dev-RG`). This role allows full VM management but not networking or role assignments.

### A4
- **Actions**: `Microsoft.Compute/virtualMachines/start/action`, `Microsoft.Compute/virtualMachines/restart/action`, `Microsoft.Compute/virtualMachines/deallocate/action`, `Microsoft.Compute/virtualMachines/read`
- **NotActions**: Leave empty (don't include delete)
- **AssignableScopes**: `["/subscriptions/{production-sub-id}"]`
- Do NOT add `Microsoft.Compute/virtualMachines/delete` to Actions.

### A5
Use the **Deny** effect. The built-in policy **"Require a tag and its value on resources"** or **"Require a tag on resources"** can be used. Assign it to the "Production-RG" scope with the tag name parameter set to "CostCenter".

### A6
**Delete lock** — Yes, modifying the replication setting will **succeed** because a Delete lock only prevents deletion; modifications are allowed.
**ReadOnly lock** — No, modifying the replication setting will **fail** because a ReadOnly lock prevents all modifications. Only read operations are allowed.

### A7
Create a **Management Group**, move all 50 subscriptions under it, then assign the **"Allowed locations"** Azure Policy at the management group scope. The policy will be inherited by all child subscriptions.

### A8
Create a **Guest user** account via Azure AD B2B invitation. By default, guest users **can**: access assigned apps, be members of groups, be assigned Azure roles. They **cannot**: enumerate directory users/groups, read all directory data, create apps, or register devices.

### A9
Azure AD enforces a **minimum of two authentication methods** for admin accounts regardless of the policy setting. Additionally, **security questions are NOT available** for admin SSPR. This is a built-in security measure that cannot be overridden.

### A10
When a subscription is transferred to a different tenant, **all RBAC role assignments, custom roles, Azure Policies, and resource locks are NOT transferred** — they are permanently deleted. The team will need to recreate all role assignments and policies in the new tenant.

### A11
Use **Azure Policy** with the **"Allowed virtual machine size SKUs"** built-in policy and set the allowed values to exclude sizes larger than Standard_D4s_v3. Azure Policy enforces *what* resources look like. RBAC controls *who* can do *what* — it's not designed for restricting resource properties.

### A12
No, deny assignments **cannot be created directly** by users. They can only be created by **Azure Blueprints** and **Azure managed applications**. If the team lead needs to block specific actions, they should use Azure Policy with a **Deny** effect instead.

### A13
**No.** Tags are **NOT inherited** from resource groups to resources by default. To auto-inherit tags, configure an Azure Policy with the **"Inherit a tag from the resource group"** policy definition using the **Modify** effect.

### A14
Use **Bulk create** in the Azure portal (Azure AD → Users → Bulk operations → Bulk create). Download the CSV template, fill in the 500 users' data, and upload the CSV. Alternatively, use **PowerShell** with `Import-Csv` and `New-AzureADUser` in a loop.

### A15
RBAC is **additive** — permissions are combined across all assignments.
- **VM in Dev-RG**: Sarah has **Contributor** permissions (Reader + Contributor = Contributor, the higher permission applies).
- **VM in Prod-RG**: Sarah has only **Reader** permissions (the Contributor role is scoped to Dev-RG and doesn't apply to Prod-RG).
