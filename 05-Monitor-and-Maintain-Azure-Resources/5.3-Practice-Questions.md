# 5.3 Practice Questions — Monitor and Maintain Azure Resources

> Scenario-based questions to test your understanding. Answers are at the bottom.

---

## Questions

### Q1 — Memory Metrics
You check Azure Monitor metrics for a VM and can't find "Available Memory Bytes". The VM has no agents installed. Why?

---

### Q2 — Alert Configuration
You need an alert that fires when CPU exceeds 80% for 5 minutes on any VM in a resource group. The ops team should get an email and SMS. What do you configure?

---

### Q3 — Activity Log Retention
Your compliance team needs Azure Activity Log data for 2 years. The Activity Log only retains data for 90 days by default. What should you do?

---

### Q4 — KQL Query
You need to find all VMs where CPU usage exceeded 90% in the last 24 hours. Write the KQL query.

---

### Q5 — VM Backup Restore
A VM's data disk is corrupted but the OS disk is fine. You don't want to restore the entire VM. What restore option should you use?

---

### Q6 — Recovery Services Vault Redundancy
You created a Recovery Services vault, configured GRS redundancy, then backed up 5 VMs. Your manager now wants to change to LRS to save costs. Can you?

---

### Q7 — Azure Site Recovery
Your company needs all production VMs in East US to be recoverable in West US within 30 minutes if the entire East US region goes down. RPO should be under 15 minutes. What should you configure?

---

### Q8 — File Recovery
A developer accidentally deleted a single file from a VM two days ago. A daily backup exists. They need just that one file, not a full VM restore. What's the fastest approach?

---

### Q9 — Soft Delete
Someone stops VM backup protection and selects "Delete backup data". The compliance team needs that data. Can it be recovered?

---

### Q10 — Azure Monitor Agent
Your team uses the legacy Log Analytics Agent (MMA). Management asks about upgrading. What are the benefits of switching to Azure Monitor Agent (AMA)?

---

## Answers

### A1
**Memory metrics are NOT available as platform metrics.** Platform metrics only include CPU, Network, and Disk I/O. To collect memory metrics, install the **Azure Monitor Agent (AMA)** and configure a **Data Collection Rule** for guest OS performance counters.

### A2
Configure:
1. **Action Group**: Email + SMS actions for the ops team
2. **Metric Alert Rule**: Resource = Resource Group scope, Metric = Percentage CPU, Aggregation = Average, Operator = Greater than, Threshold = 80, Evaluation period = 5 minutes, Frequency = 1 minute
3. Link the action group to the alert rule

### A3
Create a **Diagnostic Setting** on the subscription to send Activity Log data to:
- **Log Analytics workspace** — for querying with KQL (recommended)
- **Storage account** — for long-term archival at lower cost

Set retention to 730 days (2 years) in the Log Analytics workspace or use storage account lifecycle policies.

### A4
```kql
Perf
| where ObjectName == "Processor" and CounterName == "% Processor Time"
| where TimeGenerated > ago(24h)
| where CounterValue > 90
| summarize MaxCPU = max(CounterValue), AvgCPU = avg(CounterValue) by Computer
| order by MaxCPU desc
```

### A5
Use **Restore disks** option — restore only the data disk from the recovery point, then manually detach the corrupted disk and attach the restored disk. Alternatively, use **File Recovery** to mount the recovery point and copy specific files/folders.

### A6
**No.** Storage redundancy for a Recovery Services vault **cannot be changed after** the first backup item is registered. This setting must be configured before any backups are performed. You would need to create a new vault with LRS, re-configure backups, and decommission the old vault.

### A7
Configure **Azure Site Recovery (ASR)**:
1. Create a Recovery Services vault in **West US** (target region)
2. Enable **replication** for all production VMs from East US → West US
3. Create a **Recovery Plan** grouping VMs by tier (DB first → App → Web)
4. ASR provides RPO typically < 5 minutes (meets the <15 min requirement)
5. Perform regular **test failovers** to validate RTO meets 30-minute target
6. Configure automation scripts in the recovery plan to start dependent services

### A8
Use **File Recovery** from Azure Backup:
1. Recovery Services vault → Backup Items → Select the VM
2. Click **File Recovery**
3. Select the recovery point from 2 days ago
4. Download and run the **mount script** on any machine
5. The recovery point mounts as a drive (iSCSI)
6. Navigate to the file, copy it
7. Unmount the drive

This is much faster than restoring the entire VM.

### A9
**Yes**, if **Soft Delete** is enabled (it is by default). Deleted backup data is retained for **14 additional days** in a soft-deleted state. Go to the Recovery Services vault → Backup Items → show soft-deleted items → select the item → **Undelete**. After undeleting, resume backup protection.

### A10
Benefits of **Azure Monitor Agent (AMA)** over legacy Log Analytics Agent:
- **Data Collection Rules (DCR)**: Fine-grained control — collect only what you need, filter at source
- **Multi-homing**: Send data to multiple workspaces easily
- **Managed Identity**: Uses managed identity instead of keys (more secure)
- **Centralized configuration**: Rules apply to multiple VMs — no per-VM config
- **Future-proof**: Legacy agent is deprecated (Aug 2024); AMA is the only supported agent going forward
